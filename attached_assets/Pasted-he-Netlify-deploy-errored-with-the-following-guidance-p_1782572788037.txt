Diagnosis

The build fails during dependency installation: see line 12 and the pnpm error about a frozen lockfile at line 16.
pnpm reports the lockfile is out of date: "specifiers in the lockfile don't match specifiers in package.json" and specifically that a dependency was removed: "react-snap@^1.23.0" (see line 19). The pnpm install step then fails (line 20–line 21).
Root cause: Netlify CI runs pnpm with a frozen-lockfile by default, and your committed pnpm-lock.yaml is not in sync with package.json (a package was removed/changed locally but the lockfile was not updated and committed).
Solution

Verify package.json reflects the intended dependencies
Confirm whether react-snap was intentionally removed or should still be present. Check your repo's package.json: https://github.com/hspillers67/proforma-mvp-website/blob/main/package.json
Update and commit the lockfile (recommended)
Locally run pnpm to update the lockfile and commit it so CI can install using the frozen lockfile:
# from repo root
git pull
pnpm install    # updates pnpm-lock.yaml to match package.json
git add pnpm-lock.yaml
git commit -m "Update pnpm-lock.yaml to match package.json"
git push
This is the best fix: keep package.json and pnpm-lock.yaml in sync and push the updated lockfile.
Temporary alternative (if you cannot update the lockfile immediately)
In Netlify build settings you can allow installing with a non-frozen lockfile by passing the pnpm flag --no-frozen-lockfile. Set an environment variable in Netlify:
Key: PNPM_FLAGS
Value: --no-frozen-lockfile
Or change your build command to: pnpm install --no-frozen-lockfile && <your-build-command>
Note: this is a temporary workaround. The correct long-term fix is to commit the updated pnpm-lock.yaml.
Why this works

Netlify CI enforces frozen lockfiles by default (see the message at line 16). If your package.json and pnpm-lock.yaml differ (e.g., react-snap removed), pnpm refuses to install. Updating and committing the lockfile resolves the mismatch.
If you run into Node version issues afterwards, follow Netlify's documentation for changing Node versions: https://docs.netlify.com/configure-builds/manage-dependencies/#node-js-and-javascript
